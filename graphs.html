PATTERN 06
Breadth-First Search

Explores all neighbors at the current depth before moving deeper. Uses a queue (FIFO). Guarantees shortest path in unweighted graphs. Click a node to start BFS from it.

// CLICK A NODE TO START BFS — nodes glow as visited
Click any node to begin BFS
from collections import deque def bfs(graph, start): visited = set([start]) queue = deque([start]) order = [] while queue: node = queue.popleft() # FIFO order.append(node) for neighbor in graph[node]: if neighbor not in visited: visited.add(neighbor) queue.append(neighbor) return order # Time: O(V + E) Space: O(V) # Shortest path in unweighted graph: # track parent to reconstruct path
# BFS SHORTEST PATH (unweighted) def shortest_path(graph, src, dst): queue = deque([(src, [src])]) visited = {src} while queue: node, path = queue.popleft() if node == dst: return path for nb in graph[node]: if nb not in visited: visited.add(nb) queue.append((nb, path+[nb])) return [] # no path # BFS explores level by level → # first time we reach dst is via # the shortest path.
PATTERN 07
Depth-First Search

Goes as deep as possible before backtracking. Uses a stack (or recursion). Foundation of cycle detection, connected components, and topological sort.

# Recursive DFS def dfs(graph, node, visited=None): if visited is None: visited = set() visited.add(node) print(node) # process node for neighbor in graph[node]: if neighbor not in visited: dfs(graph, neighbor, visited) # Iterative DFS (explicit stack) def dfs_iter(graph, start): visited, stack = set(), [start] order = [] while stack: node = stack.pop() # LIFO if node not in visited: visited.add(node) order.append(node) stack.extend(graph[node]) return order
# CYCLE DETECTION (undirected graph) def has_cycle(graph, n): visited = set() def dfs(node, parent): visited.add(node) for nb in graph[node]: if nb not in visited: if dfs(nb, node): return True elif nb != parent: return True # back edge! return False for i in range(n): if i not in visited: if dfs(i, -1): return True return False # Directed graph: use WHITE/GRAY/BLACK # GRAY = in current path → cycle!
PATTERN 08
Dijkstra's Algorithm

Finds shortest paths from a source to all other nodes in a weighted graph (non-negative weights). Uses a min-heap priority queue. Greedy — always processes the nearest unvisited node.

// DIJKSTRA FROM NODE A — watch distances update
Press Play to start Dijkstra from node A
import heapq def dijkstra(graph, src): # dist[node] = shortest dist from src dist = {node: float('inf') for node in graph} dist[src] = 0 heap = [(0, src)] # (cost, node) while heap: cost, u = heapq.heappop(heap) if cost > dist[u]: continue for v, w in graph[u]: new_dist = dist[u] + w if new_dist < dist[v]: dist[v] = new_dist heapq.heappush(heap, (new_dist, v)) return dist # Time: O((V+E) log V) # Only works with non-negative weights! # Negative weights → use Bellman-Ford
# KEY INSIGHT: # When we pop a node from the min-heap, # its distance is FINAL (greedy proof). # Any other path to it must go through # a node with equal or greater distance. # RECONSTRUCT PATH: # Track parent = {} alongside dist. # When dist[v] improves, parent[v] = u # VARIANTS: # - Bidirectional Dijkstra (faster in practice) # - A* (Dijkstra + heuristic) # - Modified for k-th shortest path # COMMON INTERVIEW PATTERNS: # - Network delay time (LC 743) # - Cheapest flights (LC 787 - use Bellman) # - Path with minimum effort (LC 1631) # - Swim in rising water (LC 778)
PATTERN 09
Topological Sort

Orders nodes in a DAG so every directed edge u→v has u before v. Used for dependency resolution, course scheduling, build systems. Two approaches: Kahn's (BFS) or DFS post-order.

// KAHN'S ALGORITHM (BFS-based) — courses with prerequisites
Topo order:
Press Play to run Kahn's algorithm
# KAHN'S ALGORITHM (BFS-based) from collections import deque def topological_sort(n, edges): graph = [[] for _ in range(n)] in_degree = [0] * n for u, v in edges: graph[u].append(v) in_degree[v] += 1 # Start with all nodes that have no prereqs queue = deque(i for i in range(n) if in_degree[i]==0) order = [] while queue: u = queue.popleft() order.append(u) for v in graph[u]: in_degree[v] -= 1 if in_degree[v] == 0: queue.append(v) # Cycle detected if order incomplete return order if len(order)==n else []
# DFS-based Topological Sort def topo_dfs(graph, n): visited, stack = set(), [] def dfs(u): visited.add(u) for v in graph[u]: if v not in visited: dfs(v) stack.append(u) # post-order for i in range(n): if i not in visited: dfs(i) return stack[::-1] # reverse # Key insight: a node is added to result # AFTER all its dependencies are processed # (post-order). Reversing gives topo order. # Classic problems: # - Course Schedule (LC 207) # - Course Schedule II (LC 210) # - Alien Dictionary (LC 269)
PATTERN 10
Union-Find (Disjoint Set Union)

Tracks connected components efficiently. Two operations: find(x) — which group is x in? union(x,y) — merge groups of x and y. With path compression + union by rank: nearly O(1) amortized.

// UNION-FIND — click two nodes to union them
Click two nodes to union them
class UnionFind: def __init__(self, n): self.parent = list(range(n)) self.rank = [0] * n self.components = n def find(self, x): # Path compression if self.parent[x] != x: self.parent[x] = self.find(self.parent[x]) return self.parent[x] def union(self, x, y): px, py = self.find(x), self.find(y) if px == py: return False # same group # Union by rank if self.rank[px] < self.rank[py]: px, py = py, px self.parent[py] = px if self.rank[px] == self.rank[py]: self.rank[px] += 1 self.components -= 1 return True
# Applications: # 1. Number of Connected Components (LC 323) # 2. Redundant Connection — detect cycle (LC 684) # 3. Accounts Merge (LC 721) # 4. Number of Islands II (LC 305) # Complexity (with optimizations): # find: O(α(n)) ≈ O(1) amortized # union: O(α(n)) ≈ O(1) amortized # α = inverse Ackermann function # KEY TRICK — Weighted edges: # Store extra info in DSU (e.g. distance # to root) for problems like # "Evaluate Division" (LC 399) # Kruskal's MST uses Union-Find: # Sort edges by weight, add edge if # endpoints are in different components

Famous Graph Questions

Number of Islands (LeetCode 200)Medium

Given a 2D grid of '1' (land) and '0' (water), count the number of islands.

Approach: DFS/BFS from each unvisited '1'. Mark all connected land as visited. Each new DFS call = one island. Time O(m×n).

Key: Treat 2D grid as graph where each cell connects to its 4 neighbors.
Clone Graph (LeetCode 133)Medium

Return a deep copy of a connected undirected graph.

Approach: DFS with a hash map {original → clone}. For each node, create its clone and recursively clone all neighbors. The map handles cycles by returning existing clones.
Word Ladder (LeetCode 127)Hard

Transform beginWord to endWord changing one letter at a time, using only words in wordList. Find shortest transformation sequence.

Approach: BFS. Each word is a node; edges connect words differing by one letter. BFS gives shortest path. Optimization: bidirectional BFS from both ends.
Course Schedule II (LeetCode 210)Medium

Return order in which to take courses given prerequisites.

Approach: Topological sort (Kahn's). Build graph from prerequisites. Process nodes with in-degree 0 first. If all nodes processed, return order; else cycle exists (impossible).
Pacific Atlantic Water Flow (LeetCode 417)Medium

Find all cells that can flow water to both Pacific and Atlantic ocean.

Approach: Reverse BFS from ocean borders inward (water flows uphill in reverse). BFS from Pacific border cells, BFS from Atlantic border cells. Answer = intersection of both reachable sets.
