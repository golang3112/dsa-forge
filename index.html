PATTERN 01
Two Pointers

Use two indices that traverse the array from different positions — typically both ends moving inward. Eliminates the need for a nested loop, reducing O(n²) to O(n).

When to use: Sorted array problems where you're looking for pairs (sum, difference), or need to remove duplicates, check palindromes, or partition an array.

def two_sum_sorted(arr, target): # Two pointers from both ends left, right = 0, len(arr) - 1 while left < right: curr_sum = arr[left] + arr[right] if curr_sum == target: return [left, right] # found! elif curr_sum < target: left += 1 # need bigger else: right -= 1 # need smaller return [] # Time: O(n) Space: O(1)

Classic Two-Pointer Problems

Container With Most Water (LeetCode 11) Medium

Given n vertical lines with heights height[i], find two lines that together with the x-axis form a container with maximum water.

Approach: Start with left=0, right=n-1. Area = min(h[l],h[r]) × (r−l). Move the pointer with the smaller height inward — moving the larger one can only decrease area.

Key insight: Width decreases each step, so we must increase height to find better candidates.
3Sum (LeetCode 15) Medium

Find all unique triplets in the array which sum to zero.

Approach: Sort the array. Fix one element at index i, then use two pointers (i+1, n-1) to find pairs summing to -arr[i]. Skip duplicates by checking arr[i]==arr[i-1].

Time: O(n²) — sorting O(n log n) + two-pointer O(n²)
Trapping Rain Water (LeetCode 42) Hard

Given n non-negative integers representing an elevation map, compute how much water it can trap after raining.

Approach: Two pointers l=0, r=n-1. Track maxLeft and maxRight. Water at position i = min(maxLeft, maxRight) - height[i]. Move the side with smaller max height.

Why it works: The limiting factor for water at any cell is the shorter of the two max heights seen from each side.
PATTERN 02
Sliding Window

Maintain a window of elements that expands (right pointer) and contracts (left pointer) to satisfy a constraint. Turns O(n²) subarray problems into O(n).

When to use: "Longest/shortest subarray/substring with property X" — contiguous subarray, running sum or frequency count, substring with k distinct chars, etc.

def max_sum_subarray(arr, k): n = len(arr) window_sum = sum(arr[:k]) max_sum = window_sum # Slide the window for i in range(k, n): # Add right element, remove left window_sum += arr[i] - arr[i - k] max_sum = max(max_sum, window_sum) return max_sum # Time: O(n) Space: O(1) # Key: avoid recomputing sum from scratch!
# VARIABLE WINDOW — Longest substring k distinct chars def longest_k_distinct(s, k): freq = {} left = max_len = 0 for right in range(len(s)): freq[s[right]] = freq.get(s[right], 0) + 1 while len(freq) > k: # shrink freq[s[left]] -= 1 if freq[s[left]] == 0: del freq[s[left]] left += 1 max_len = max(max_len, right - left + 1) return max_len

Classic Sliding Window Problems

Longest Substring Without Repeating Characters (LeetCode 3) Medium

Find the length of the longest substring without repeating characters.

Approach: Variable sliding window. Use a set to track chars in window. When a duplicate is found, shrink window from left until the duplicate is removed.

Example: "abcabcbb" → window expands: a,ab,abc → 'a' seen again → shrink to bc → expand bcab... max=3
Minimum Window Substring (LeetCode 76) Hard

Given strings s and t, return the minimum window in s that contains all characters in t.

Approach: Expand right until window contains all t chars. Then shrink left while still valid, recording minimum. Two frequency maps — need and have counters.

Time: O(n) — each char visited at most twice (once by right, once by left)
PATTERN 03
Prefix Sum

Precompute cumulative sums so any range query [l, r] can be answered in O(1) instead of O(n). The sum of arr[l..r] = prefix[r+1] − prefix[l].

When to use: Repeated range sum queries, subarray sum equals k, equilibrium index, number of subarrays with given sum.

PATTERN 04
Kadane's Algorithm

Find the maximum sum contiguous subarray in O(n). The key insight: at each position, either extend the previous subarray or start a new one — whichever is larger.

def kadane(arr): max_sum = curr_sum = arr[0] for i in range(1, len(arr)): # Extend or start fresh curr_sum = max(arr[i], curr_sum + arr[i]) max_sum = max(max_sum, curr_sum) return max_sum # Time: O(n) Space: O(1) # Classic: LeetCode 53 - Maximum Subarray # Variant: return subarray indices def kadane_indices(arr): max_sum = curr_sum = arr[0] start = end = temp_start = 0 for i in range(1, len(arr)): if arr[i] > curr_sum + arr[i]: curr_sum = arr[i]; temp_start = i else: curr_sum += arr[i] if curr_sum > max_sum: max_sum = curr_sum start, end = temp_start, i return max_sum, start, end

Famous Interview Questions

Two Sum (LeetCode 1)Easy

Return indices of two numbers that add up to target.

Pattern: Hash map. Store each value → index. For each element, check if complement (target - x) exists in map. O(n) time, O(n) space.
Best Time to Buy and Sell Stock (LeetCode 121)Easy

Find max profit from one buy and one sell transaction.

Pattern: Track min price seen so far. Profit = current - min_price. Keep updating max_profit. O(n) single pass.
Product of Array Except Self (LeetCode 238)Medium

Return array where output[i] = product of all elements except arr[i]. No division allowed.

Pattern: Two-pass prefix/suffix products. Left pass: output[i] = product of all left. Right pass: multiply by suffix products. O(n) time, O(1) extra space.
Subarray Sum Equals K (LeetCode 560)Medium

Count number of subarrays that sum to k.

Pattern: Prefix sum + hash map. If prefix[j] - prefix[i] = k, then subarray [i+1, j] sums to k. Track count of each prefix sum seen so far.
Longest Repeating Character Replacement (LeetCode 424)Medium

With at most k replacements, find length of longest substring with one repeated char.

Pattern: Sliding window. Track max frequency char in window. Window is invalid if (window_size - max_freq) > k. Shrink left when invalid.
Spiral Matrix (LeetCode 54)Medium

Return all elements of matrix in spiral order.

Pattern: Simulation with four boundary pointers (top, bottom, left, right). Traverse layer by layer, shrinking boundaries after each pass. O(m×n).

Binary Search

The most powerful O(log n) technique in interviews. Classic search, rotated arrays, first & last position, and the answer-space template — all step-by-step.

PATTERN — CLASSIC BINARY SEARCH
Classic Binary Search

On a sorted array, check the middle element. If too small, search right half; if too large, search left half. Eliminates half the search space each step → O(log n).

Template: lo=0, hi=n-1. While lo≤hi: mid=(lo+hi)//2. Compare arr[mid] with target and halve. Beware: use lo + (hi-lo)//2 instead of (lo+hi)//2 to avoid integer overflow in other languages.

def binary_search(arr, target): lo, hi = 0, len(arr) - 1 while lo <= hi: mid = lo + (hi - lo) // 2 # safe mid if arr[mid] == target: return mid # found! elif arr[mid] < target: lo = mid + 1 # search right else: hi = mid - 1 # search left return -1 # not found # Each step: search space halves # n → n/2 → n/4 → … → 1 # Steps needed: log₂(n) # For n=1,000,000: only ~20 steps!
# PYTHON BUILT-IN: bisect module import bisect arr = [1,3,5,7,9,11,13] # bisect_left: leftmost insert position i = bisect.bisect_left(arr, 7) # → 3 # bisect_right: rightmost insert position j = bisect.bisect_right(arr, 7) # → 4 # Check if target exists: if i < len(arr) and arr[i] == 7: print("Found at", i) # Count occurrences of target: # count = bisect_right(arr,t) - bisect_left(arr,t)
PATTERN — BOUNDARY SEARCH
First & Last Position

When duplicates exist, find the leftmost or rightmost occurrence. The trick: even when you find the target, keep searching in the relevant direction instead of returning immediately.

# Combined solution (LC 34) def search_range(nums, target): return [find_first(nums, target), find_last(nums, target)] # KEY INSIGHT: # Normal BS: return immediately on match. # First pos: on match → hi = mid-1 # (keep searching left) # Last pos: on match → lo = mid+1 # (keep searching right) # # Both still run in O(log n). # Run twice → O(2 log n) = O(log n). # # ALTERNATIVE: use bisect # first = bisect_left(arr, target) # last = bisect_right(arr, target) - 1 # if first==len or arr[first]!=target: # return [-1,-1]

PATTERN — ANSWER SPACE BINARY SEARCH
Binary Search on the Answer

Don't search the array — search the answer space. If answers form a monotone feasibility function (below threshold = infeasible, above = feasible), binary search for the boundary. Converts O(n²) to O(n log n).

Template: lo = min possible answer, hi = max possible answer. While lo < hi: mid = (lo+hi)//2. If feasible(mid): hi=mid. Else: lo=mid+1. Return lo.

Famous Binary Search Questions

Binary Search (LeetCode 704)Easy

Classic binary search on a sorted array.

Template: lo=0, hi=n-1. While lo≤hi: mid=(lo+hi)//2. Three-way compare and halve. Return -1 if not found. O(log n).
Find Minimum in Rotated Sorted Array (LeetCode 153)Medium

Find the minimum in a rotated sorted array without duplicates.

Approach: If arr[mid] > arr[hi], the minimum is in the right half (lo=mid+1). Else hi=mid. When lo==hi, that's the minimum. O(log n). Key insight: minimum is always in the "unsorted" half.
Search a 2D Matrix (LeetCode 74)Medium

Search in an m×n matrix where each row is sorted and first element of each row is greater than the last of the previous row.

Trick: Treat the 2D matrix as a flat sorted 1D array. Index i maps to matrix[i//n][i%n]. Binary search on 0..m*n-1. O(log(m×n)).
Koko Eating Bananas (LeetCode 875)Medium

Find minimum eating speed k to finish all piles in h hours.

Answer-space BS: lo=1, hi=max(piles). For each mid (speed), check if total ceil(p/k) hours ≤ h. If yes, try slower (hi=mid). If no, go faster (lo=mid+1). O(n log m).
Median of Two Sorted Arrays (LeetCode 4)Hard

Find the median of two sorted arrays in O(log(m+n)).

Binary search on partition: Binary search on the shorter array's partition point. Ensure left halves of both arrays together have (m+n)//2 elements. Check that maxLeft ≤ minRight across both arrays. Adjust partition until condition satisfied.
Time Based Key-Value Store (LeetCode 981)Medium

Design a key-value store that retrieves values at or before a given timestamp.

BS on timestamps: Store (timestamp, value) list per key. On get, binary search for largest timestamp ≤ given timestamp using bisect_right. O(log n) per query.
Split Array Largest Sum (LeetCode 410)Hard

Split array into m subarrays minimising the largest subarray sum.

Answer-space BS: lo=max(nums), hi=sum(nums). For each mid (capacity), greedily count partitions needed. If count ≤ m: hi=mid (feasible). Else: lo=mid+1. O(n log(sum)).

PATTERN 11 — MEMOIZATION
Top-Down DP (Memoization)

Cache results of recursive calls so each subproblem is solved only once. Fibonacci without memoization is O(2ⁿ); with memo it's O(n). The cache turns exponential into linear.

The key question: Does the problem have overlapping subproblems (same sub-call made multiple times) and optimal substructure (optimal solution built from optimal subsolutions)? If yes → DP.

// FIB(6) CALL TREE — without vs with memoization
Choose a mode above
# WITHOUT memo — O(2ⁿ), recomputes same values def fib_naive(n): if n <= 1: return n return fib_naive(n-1) + fib_naive(n-2) # fib(5) calls fib(3) TWICE, # fib(2) THREE times, fib(1) FIVE times! # WITH memo — O(n), each value once def fib_memo(n, memo={}): if n in memo: return memo[n] if n <= 1: return n memo[n] = fib_memo(n-1, memo) + fib_memo(n-2, memo) return memo[n]
# BOTTOM-UP tabulation — same O(n) def fib_dp(n): if n <= 1: return n dp = [0] * (n+1) dp[1] = 1 for i in range(2, n+1): dp[i] = dp[i-1] + dp[i-2] return dp[n] # Space-optimised — O(1) space def fib_opt(n): a, b = 0, 1 for _ in range(n): a, b = b, a+b return a # Top-down vs Bottom-up: # Top-down = recursion + cache (natural) # Bottom-up = iterative table (faster)
PATTERN 12 — 0/1 KNAPSACK
0/1 Knapsack

Given items with weights and values, maximise value in a knapsack of capacity W. Each item either taken (1) or not (0). The DP table fills: dp[i][w] = max value using first i items with capacity w.

// 0/1 KNAPSACK TABLE — watch cells fill row by row
Items: [(wt=2,val=6), (wt=2,val=10), (wt=3,val=12)]  |  Capacity: 5
Press Play to fill the DP table
def knapsack(weights, values, W): n = len(weights) # dp[i][w] = max val using items 0..i-1 # with capacity w dp = [[0]*(W+1) for _ in range(n+1)] for i in range(1, n+1): for w in range(W+1): # Don't take item i dp[i][w] = dp[i-1][w] # Take item i (if it fits) if weights[i-1] <= w: take = dp[i-1][w-weights[i-1]] \ + values[i-1] dp[i][w] = max(dp[i][w], take) return dp[n][W] # O(n×W) time & space
# RECONSTRUCT which items were taken def trace_items(dp, weights, values, W): items = [] i, w = len(weights), W while i > 0 and w > 0: if dp[i][w] != dp[i-1][w]: # Item i was included items.append(i-1) w -= weights[i-1] i -= 1 return items[::-1] # SPACE OPTIMISATION — O(W) space # Use 1D array, iterate w in reverse! def knapsack_1d(weights, values, W): dp = [0] * (W+1) for i in range(len(weights)): for w in range(W, weights[i]-1, -1): dp[w] = max(dp[w], dp[w-weights[i]] + values[i]) return dp[W]
PATTERN 13 — LCS
Longest Common Subsequence

Find the longest sequence of characters that appears in both strings (not necessarily contiguous). Foundation of diff tools, DNA alignment, and edit distance. DP table: dp[i][j] = LCS length of s1[0..i] and s2[0..j].

// LCS DP TABLE — "ABCBDAB" vs "BDCABA"
S1: "ABCBDAB"  |  S2: "BDCABA"
Press Play to fill the LCS table
def lcs(s1, s2): m, n = len(s1), len(s2) # dp[i][j] = LCS of s1[:i] and s2[:j] dp = [[0]*(n+1) for _ in range(m+1)] for i in range(1, m+1): for j in range(1, n+1): if s1[i-1] == s2[j-1]: # Characters match! Extend LCS dp[i][j] = dp[i-1][j-1] + 1 else: # Take best of skip either char dp[i][j] = max(dp[i-1][j], dp[i][j-1]) return dp[m][n] # O(m×n) time & space
# RECONSTRUCT the actual LCS string def get_lcs(dp, s1, s2): i, j = len(s1), len(s2) result = [] while i > 0 and j > 0: if s1[i-1] == s2[j-1]: result.append(s1[i-1]) i -= 1; j -= 1 elif dp[i-1][j] > dp[i][j-1]: i -= 1 else: j -= 1 return ''.join(reversed(result)) # RELATED PROBLEMS: # Edit Distance: dp[i][j] = min ops # to convert s1[:i] to s2[:j] # Longest Palindromic Subsequence: # LCS(s, reverse(s)) # Shortest Common Supersequence: # len(s1)+len(s2) - LCS(s1,s2)
PATTERN 14 — COIN CHANGE
Coin Change

Find the minimum number of coins to make amount. Each coin can be used unlimited times (unbounded knapsack variant). dp[a] = min coins to make amount a. Build from amount 0 up to target.

// COIN CHANGE DP ARRAY — coins [1,3,4], amount 6
Coins: 134   Amount: 6
Press Play
def coin_change(coins, amount): # dp[a] = min coins to make amount a dp = [float('inf')] * (amount + 1) dp[0] = 0 # 0 coins to make amount 0 for a in range(1, amount + 1): for coin in coins: if coin <= a: dp[a] = min(dp[a], dp[a - coin] + 1) return dp[amount] if dp[amount] != float('inf') else -1 # Time: O(amount × len(coins)) # Space: O(amount) # Key insight: for each amount a, # try subtracting every coin and use # the best previously computed result.
# COIN CHANGE II — count ways (LC 518) def coin_change_ways(coins, amount): dp = [0] * (amount + 1) dp[0] = 1 # 1 way to make 0 # Iterate coins first (avoid duplicates) for coin in coins: for a in range(coin, amount + 1): dp[a] += dp[a - coin] return dp[amount] # DIFFERENCE from min-coins: # Min coins: iterate amounts outer # Count ways: iterate coins outer # (prevents counting same combo twice) # Example: coins=[1,2], amount=3 # Ways: {1+1+1, 1+2, 2+1} → wait, # 1+2 and 2+1 are same → outer=coin # gives 2 unique ways: [1,1,1],[1,2]

Famous DP Questions

Climbing Stairs (LeetCode 70)Easy

Count ways to reach n-th stair taking 1 or 2 steps.

Pattern: Fibonacci DP. dp[i] = dp[i-1] + dp[i-2]. It's Fibonacci starting with dp[1]=1, dp[2]=2. Space-optimise to O(1) with two variables.
House Robber (LeetCode 198)Medium

Rob houses in a row — can't rob adjacent houses. Maximise total.

Pattern: dp[i] = max(dp[i-1], dp[i-2] + nums[i]). Either skip this house (take dp[i-1]) or rob it (dp[i-2] + current). O(n) time, O(1) space.
Longest Increasing Subsequence (LeetCode 300)Medium

Find length of longest strictly increasing subsequence.

O(n²) DP: dp[i] = max LIS ending at index i. For each i, check all j < i where arr[j] < arr[i].

O(n log n) patience sort: Maintain tails array. Binary search for position to place each element. Length of tails = LIS length.
Edit Distance (LeetCode 72)Hard

Minimum operations (insert/delete/replace) to convert word1 to word2.

Pattern: LCS-style 2D DP. dp[i][j] = min ops to convert s1[:i] to s2[:j]. If chars match: dp[i-1][j-1]. Else: 1 + min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1]) for delete/insert/replace.
Partition Equal Subset Sum (LeetCode 416)Medium

Can array be partitioned into two subsets with equal sum?

Pattern: 0/1 Knapsack. Target = total_sum/2. dp[j] = can we make sum j? For each num, update dp in reverse: dp[j] |= dp[j-num]. Return dp[target].
Word Break (LeetCode 139)Medium

Can string s be segmented into words from wordDict?

Pattern: 1D DP. dp[i] = can s[:i] be segmented? For each i, check all j < i: if dp[j] and s[j:i] in dict → dp[i] = True. Time O(n² × m) where m = avg word length.

PATTERN 06
Breadth-First Search

Explores all neighbors at the current depth before moving deeper. Uses a queue (FIFO). Guarantees shortest path in unweighted graphs. Click a node to start BFS from it.

PATTERN — FLOYD'S CYCLE DETECTION
Floyd's Tortoise & Hare

Two pointers: slow moves 1 step, fast moves 2 steps. If there's a cycle, they'll meet inside it. If no cycle, fast reaches null. Uses O(1) space — no hash set needed.

Why they meet: Once inside a cycle, fast gains 1 step per iteration on slow. The gap closes to 0 — they must meet. The meeting point can locate the cycle start with a clever math trick.

PATTERN 06
Binary Search Tree

A BST satisfies: all nodes in left subtree < root < all nodes in right subtree. This property enables O(log n) search, insert, and delete on balanced trees.

BST invariant: left.val < node.val < right.val for every node. In-order traversal of a BST gives sorted output — that's why BST is so powerful for range queries.

Key Tree Patterns

Pattern 1 — Tree DP (post-order): Most "path" problems compute a value for each subtree in post-order (both children first), then combine. Height, diameter, max path sum all use this.

# MAX PATH SUM — Tree DP pattern (LeetCode 124) def max_path_sum(root): self.max_sum = float('-inf') def dfs(node): if not node: return 0 left = max(dfs(node.left), 0) # ignore negative branches right = max(dfs(node.right), 0) # Path through current node (can't extend upward AND go both ways) self.max_sum = max(self.max_sum, node.val + left + right) # Return best single-direction path for parent return node.val + max(left, right) dfs(root); return self.max_sum

Pattern 2 — Lowest Common Ancestor (LCA): DFS returns node if found, None otherwise. If both children return non-null, current node is LCA.

# LOWEST COMMON ANCESTOR — (LeetCode 236) def lca(root, p, q): if not root or root == p or root == q: return root left = lca(root.left, p, q) right = lca(root.right, p, q) # Both sides found something → current node is LCA if left and right: return root return left if left else right

Pattern 3 — Tree Diameter: At each node, diameter through it = left_height + right_height. Global max is the answer.
