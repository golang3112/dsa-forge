PATTERN — REVERSAL
Reverse a Linked List

Iteratively reverse a singly linked list using three pointers: prev, curr, and next. Each step detaches curr from the original chain and points it backward.

// REVERSAL — watch prev (green) and curr (purple) advance
Press Play to begin reversal
# Iterative reversal — O(n) time, O(1) space def reverse_list(head): prev = None curr = head while curr: next_node = curr.next # save next curr.next = prev # reverse link prev = curr # advance prev curr = next_node # advance curr return prev # new head # Recursive version def reverse_recursive(head): if not head or not head.next: return head new_head = reverse_recursive(head.next) head.next.next = head head.next = None return new_head
# REVERSE between positions m and n (LC 92) def reverse_between(head, m, n): dummy = ListNode(0) dummy.next = head prev = dummy # Move prev to position m-1 for _ in range(m - 1): prev = prev.next curr = prev.next for _ in range(n - m): next_n = curr.next curr.next = next_n.next next_n.next = prev.next prev.next = next_n return dummy.next # KEY: dummy node simplifies # edge cases (reversing from head)
PATTERN — FLOYD'S CYCLE DETECTION
Floyd's Tortoise & Hare

Two pointers: slow moves 1 step, fast moves 2 steps. If there's a cycle, they'll meet inside it. If no cycle, fast reaches null. Uses O(1) space — no hash set needed.

Why they meet: Once inside a cycle, fast gains 1 step per iteration on slow. The gap closes to 0 — they must meet. The meeting point can locate the cycle start with a clever math trick.

// FLOYD'S — slow (blue) and fast (yellow) pointers
Press Play
# Detect cycle — O(n) time, O(1) space def has_cycle(head): slow = fast = head while fast and fast.next: slow = slow.next fast = fast.next.next if slow == fast: return True return False # Find cycle start node (LC 142) def cycle_start(head): slow = fast = head # Phase 1: find meeting point while fast and fast.next: slow = slow.next fast = fast.next.next if slow == fast: break else: return None # no cycle # Phase 2: move one pointer to head slow = head while slow != fast: slow = slow.next fast = fast.next return slow # cycle start!
# WHY PHASE 2 WORKS (math): # Let: # F = distance head → cycle start # C = cycle length # h = distance start → meeting point # # At meeting: slow travelled F+h # fast travelled F+h+C # fast = 2×slow # → F+h+C = 2(F+h) # → C-h = F # → F = C-h # # So moving one pointer to head and # advancing both 1-step will meet # at the CYCLE START after F steps.
PATTERN — TWO POINTERS
Find the Middle Node

Use slow/fast pointers: slow moves 1 step, fast moves 2. When fast reaches the end, slow is at the middle. One-pass O(n), O(1) space — no need to count length first.

// FIND MIDDLE — slow (blue) and fast (yellow)
Press Play
def find_middle(head): slow = fast = head while fast and fast.next: slow = slow.next # 1 step fast = fast.next.next # 2 steps return slow # middle node # For even-length list, returns SECOND middle. # To get first middle: while fast.next and fast.next.next # Applications: # - Reverse second half (palindrome check) # - Merge sort on linked list # - Find if palindrome: find mid → reverse → compare
PATTERN — MERGE
Merge Two Sorted Lists

Compare front nodes of two sorted lists, attach the smaller to result, advance that pointer. Use a dummy head to avoid edge cases. Foundation of merge sort on linked lists.

// MERGE — list1 (blue) and list2 (yellow) merge into result (green)
Press Play
# Merge two sorted lists — O(n+m) def merge_sorted(l1, l2): dummy = ListNode(0) # sentinel head curr = dummy while l1 and l2: if l1.val <= l2.val: curr.next = l1 l1 = l1.next else: curr.next = l2 l2 = l2.next curr = curr.next # Attach remaining nodes curr.next = l1 if l1 else l2 return dummy.next
# MERGE K SORTED LISTS (LC 23) import heapq def merge_k_lists(lists): heap = [] # Push first node of each list for i, node in enumerate(lists): if node: heapq.heappush(heap, (node.val, i, node)) dummy = curr = ListNode(0) while heap: val, i, node = heapq.heappop(heap) curr.next = node curr = curr.next if node.next: heapq.heappush(heap, (node.next.val, i, node.next)) return dummy.next # Time: O(N log k) — N total nodes, k lists

Famous Linked List Questions

Reverse Linked List (LeetCode 206)Easy

Reverse a singly linked list.

Iterative: prev=None, curr=head. Save next, reverse link, advance both. Return prev.
Recursive: Recurse to end, reverse links on the way back up. O(n) time, O(n) stack space.
Linked List Cycle II (LeetCode 142)Medium

Find the node where the cycle begins.

Floyd's two-phase: Phase 1 finds meeting point inside cycle. Phase 2 moves one pointer to head, advance both by 1 until they meet — that's the cycle start (proof via math: F = C-h).
Palindrome Linked List (LeetCode 234)Easy

Check if a linked list is a palindrome in O(n) time and O(1) space.

3-step: (1) Find middle with slow/fast. (2) Reverse second half in place. (3) Compare first and reversed second half. Optionally restore the list afterward.
Reorder List (LeetCode 143)Medium

Reorder L0→L1→…→Ln to L0→Ln→L1→Ln-1→L2→…

3-step: (1) Find middle. (2) Reverse second half. (3) Merge two halves by alternating nodes. Each step is O(n), O(1) space — three linked list fundamentals combined.
LRU Cache (LeetCode 146)Medium

Design a data structure that follows Least Recently Used cache policy.

Doubly linked list + hash map. Map gives O(1) access. DLL gives O(1) move-to-front and evict-tail. Most recently used at head, least recently used at tail. Classic system design + DS combo.
