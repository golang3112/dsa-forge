PATTERN 05
Tree Traversals

DFS visits nodes depth-first (pre/in/post-order). BFS visits level by level using a queue. Understanding which traversal to use is the key to most tree problems.

# Pre-order: Root → Left → Right def preorder(node): if not node: return visit(node) # Root FIRST preorder(node.left) preorder(node.right) # In-order: Left → Root → Right # Gives SORTED output for a BST! def inorder(node): if not node: return inorder(node.left) visit(node) # Root MIDDLE inorder(node.right) # Post-order: Left → Right → Root # Used for deletion, height calculation def postorder(node): if not node: return postorder(node.left) postorder(node.right) visit(node) # Root LAST
# BFS: Level-order using a queue from collections import deque def bfs(root): if not root: return queue = deque([root]) result = [] while queue: node = queue.popleft() result.append(node.val) if node.left: queue.append(node.left) if node.right: queue.append(node.right) return result # Level-by-level BFS def level_order(root): queue = deque([root]) levels = [] while queue: level = [] for _ in range(len(queue)): node = queue.popleft() level.append(node.val) if node.left: queue.append(node.left) if node.right: queue.append(node.right) levels.append(level) return levels
PATTERN 06
Binary Search Tree

A BST satisfies: all nodes in left subtree < root < all nodes in right subtree. This property enables O(log n) search, insert, and delete on balanced trees.

BST invariant: left.val < node.val < right.val for every node. In-order traversal of a BST gives sorted output — that's why BST is so powerful for range queries.

Enter a value and click Insert or Search
class BST: def insert(self, root, val): if not root: return TreeNode(val) if val < root.val: root.left = self.insert(root.left, val) else: root.right = self.insert(root.right, val) return root def search(self, root, val): if not root: return None if val == root.val: return root if val < root.val: return self.search(root.left, val) return self.search(root.right, val) # Time: O(h) where h = height # Balanced BST: h = O(log n) # Worst case (sorted input): h = O(n)
def delete(root, val): if not root: return None if val < root.val: root.left = delete(root.left, val) elif val > root.val: root.right = delete(root.right, val) else: # found node to delete if not root.left: return root.right if not root.right: return root.left # Two children: replace with inorder successor successor = get_min(root.right) root.val = successor.val root.right = delete(root.right, successor.val) return root def get_min(node): while node.left: node = node.left return node

Key Tree Patterns

Pattern 1 — Tree DP (post-order): Most "path" problems compute a value for each subtree in post-order (both children first), then combine. Height, diameter, max path sum all use this.

# MAX PATH SUM — Tree DP pattern (LeetCode 124) def max_path_sum(root): self.max_sum = float('-inf') def dfs(node): if not node: return 0 left = max(dfs(node.left), 0) # ignore negative branches right = max(dfs(node.right), 0) # Path through current node (can't extend upward AND go both ways) self.max_sum = max(self.max_sum, node.val + left + right) # Return best single-direction path for parent return node.val + max(left, right) dfs(root); return self.max_sum

Pattern 2 — Lowest Common Ancestor (LCA): DFS returns node if found, None otherwise. If both children return non-null, current node is LCA.

# LOWEST COMMON ANCESTOR — (LeetCode 236) def lca(root, p, q): if not root or root == p or root == q: return root left = lca(root.left, p, q) right = lca(root.right, p, q) # Both sides found something → current node is LCA if left and right: return root return left if left else right

Pattern 3 — Tree Diameter: At each node, diameter through it = left_height + right_height. Global max is the answer.

Famous Tree Questions

Invert Binary Tree (LeetCode 226)Easy

Mirror the tree — swap left and right children at every node.

Approach: Simple DFS. At each node, swap left and right, then recurse. Or BFS: process each node and swap. Time O(n).
Validate Binary Search Tree (LeetCode 98)Medium

Check if a binary tree is a valid BST.

Approach: Pass min/max bounds down. At each node, value must be in (min, max). Left subtree max = current value, right subtree min = current value. Common mistake: checking only parent, not full range.
Binary Tree Maximum Path Sum (LeetCode 124)Hard

Find path with maximum sum (path can start and end at any node).

Approach: Post-order DFS. At each node, compute max contribution going left or right (clamped to 0 if negative). Update global max with left + node + right. Return node + max(left, right) to parent.
Serialize and Deserialize Binary Tree (LeetCode 297)Hard

Convert tree to string and back.

Approach: BFS/pre-order serialization. Use "null" markers for missing nodes. Deserialization rebuilds using a queue/index to track position. Pre-order is simplest: root,left,right with null markers.
Kth Smallest Element in BST (LeetCode 230)Medium

Find the kth smallest element in a BST.

Approach: In-order traversal gives sorted order. Count nodes visited — return at the kth one. Iterative in-order with a stack is O(H+k) where H is height.
