Stack & Queue

Monotonic stack, valid parentheses, sliding window maximum — the patterns that turn O(n²) into O(n) with a simple data structure.

DATA STRUCTURE
Stack (LIFO) & Queue (FIFO)

Stack: Last-In-First-Out — push to top, pop from top. Queue: First-In-First-Out — enqueue at back, dequeue from front. Both O(1) for core operations.

PATTERN — STACK MATCHING
Valid Parentheses

Use a stack to match opening brackets with closing ones. Push open brackets; when a close bracket is encountered, pop and check if it matches. Stack empty at the end = valid.

PATTERN — MONOTONIC STACK
Monotonic Stack

A stack that is kept in monotonically increasing or decreasing order. Pop elements that violate the order — the popping event answers "next greater element" queries in O(n) total.

Key insight: When you pop an element because the current element is larger, the current element IS the "next greater element" for the popped one. This answers all NGE queries in a single pass.

# NEXT GREATER ELEMENT — O(n) def next_greater(arr): n = len(arr) result = [-1] * n # default: no NGE stack = [] # stores INDICES for i in range(n): # Pop elements smaller than arr[i] while stack and arr[stack[-1]] < arr[i]: idx = stack.pop() result[idx] = arr[i] # arr[i] is NGE! stack.append(i) return result # Each element pushed/popped at most once # → O(n) total, not O(n²)
# DAILY TEMPERATURES (LC 739) def daily_temperatures(T): n = len(T) result = [0] * n stack = [] # monotonic decreasing for i in range(n): while stack and T[stack[-1]] < T[i]: j = stack.pop() result[j] = i - j # days to wait stack.append(i) return result # LARGEST RECTANGLE IN HISTOGRAM (LC 84) # Monotonic increasing stack. # For each bar, pop bars taller than it. # Width = i - stack[-1] - 1 # Area = height × width
PATTERN — MONOTONIC DEQUE
Sliding Window Maximum

Find the maximum in each window of size k. Naïve is O(nk). With a monotonic deque (always keeps indices of potentially-max elements), it's O(n) — front is always the window maximum.

// SLIDING WINDOW MAX — k=3, deque shown below array

from collections import deque def sliding_window_max(nums, k): dq = deque() # stores INDICES result = [] for i, n in enumerate(nums): # Remove out-of-window indices while dq and dq[0] < i - k + 1: dq.popleft() # Remove smaller elements from back while dq and nums[dq[-1]] <= n: dq.pop() dq.append(i) # Front is always the window max if i >= k - 1: result.append(nums[dq[0]]) return result # O(n) total
# WHY THIS WORKS: # The deque is monotonically decreasing. # Any element smaller than the new element # can NEVER be a future window max # (new element is larger AND newer) → remove. # # Front of deque = index of window max. # When front goes out of window → remove. # # Example: nums=[1,3,1,2,0,5], k=3 # i=0: deque=[0] # i=1: 3>1, pop 0 → deque=[1] # i=2: deque=[1,2], window[0..2] max=3 # i=3: 2>1, pop 2 → deque=[1,3], max=3 # i=4: deque=[1,3,4], max=2 # i=5: 5>all, pop all → deque=[5], max=5

Famous Stack & Queue Questions

Min Stack (LeetCode 155)Medium

Design a stack supporting push, pop, top, and getMin in O(1).

Approach: Maintain two stacks — main stack and min stack. Min stack stores the current minimum at each level. Push to min stack only when new val ≤ current min. Pop from both when popping main stack.
Largest Rectangle in Histogram (LeetCode 84)Hard

Find the largest rectangle in a histogram.

Monotonic increasing stack: For each bar, pop all taller bars when current bar is shorter. For each popped bar, width = current_index − new_stack_top − 1. Area = height × width. Track maximum. Add sentinel 0 at the end.
Decode String (LeetCode 394)Medium

Decode strings like "3[a2[bc]]" → "abcbcabcbcabcbc".

Stack-based: Maintain stacks for current string and current count. When '[' is hit, push both. When ']' is hit, pop count and prev_string, append current string × count to prev. O(n × maxCount).
Basic Calculator II (LeetCode 227)Medium

Evaluate string expression with +, -, *, /.

Stack + sign tracking: Parse numbers. On +/−, push ±number to stack. On ×/÷, pop top and push result. Sum the stack at the end. Key: multiplication/division take priority by consuming the previous number immediately.
